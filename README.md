## What is a GTM Engineer?

A **GTM Engineer** automates go-to-market with code + AI — outbound, enrichment, lead scoring, routing, and revenue workflows. They sit between marketing/sales and engineering. It's the technical evolution of the SDR/marketer, and it's one of the most in-demand AI-native roles because almost no one can do it well yet.

---

## The roadmap

### Stage 0 — Foundations
- Spreadsheets → SQL (query your own data).
- Basic scripting (Python or JS) + how APIs work.
- Understand a CRM (HubSpot/Salesforce) data model.

### Stage 1 — The GTM stack
- **Clay** (or similar) for enrichment + workflows — learn it deeply.
- Data enrichment + waterfall providers.
- Email infrastructure, deliverability, sequencing tools.

### Stage 2 — Automation + AI
- Connect tools with APIs / webhooks / Zapier-style automation.
- Use LLMs for **personalized** outbound at scale (signals → message).
- Lead scoring + routing logic.
- Scraping + signal collection (job changes, funding, tech stack).

### Stage 3 — Build real workflows
- Ship 2–3 documented revenue workflows (enrichment, outbound, routing).
- Show **results** — reply rates, meetings booked, time saved.
- See [projects-to-land-an-ai-job](https://github.com/landedjobs/projects-to-land-an-ai-job) (GTM section).

### Stage 4 — Interview & land
- Be ready to whiteboard a workflow end-to-end and talk metrics.
- Build public proof (share your builds on LinkedIn/X).
- **Get referred** — the GTM world runs on warm intros.
- Prep with mock interviews on [Landed](https://landed.b100x.ai).
